# Sprint Update Template - Growth Sprint Presentation

Use this template to generate Growth Sprint Presentation slides from Sprint Planning Meeting notes and other relevant documents.

## Template Structure

Each team contributes 2-3 slides to the Growth Sprint Presentation:

### Slide 1: What Was Done This Sprint

```
# Slide 1: What Was Done This Sprint - [Team Name]

## Sprint #[Sprint Number]: [Start Date] - [End Date]

### Completed Work

* **[Initiative/Feature 1]**
  * [Brief description of what was completed]
  * [Key metrics or results if available]
  
* **[Initiative/Feature 2]**
  * [Brief description of what was completed]
  * [Key metrics or results if available]

### Key Deliverables

* [Deliverable 1]
* [Deliverable 2]
* [Deliverable 3]
```

### Slide 2: What's Coming Up Next Sprint

```
# Slide 2: What's Coming Up Next Sprint - [Team Name]

## Sprint #[Next Sprint Number]: [Start Date] - [End Date]

### Sprint Goal

[Sprint goal statement]

### Planned Work

* **[Initiative/Feature 1]**
  * [Brief description of what will be worked on]
  * [Expected outcomes or milestones]
  
* **[Initiative/Feature 2]**
  * [Brief description of what will be worked on]
  * [Expected outcomes or milestones]

### Focus Areas

* [Focus area 1]
* [Focus area 2]
```

### Slide 3: Demo (Optional)

```
# Slide 3: Demo - [Team Name]

## [Feature/Initiative Name] Demo

### What We're Showing

[Brief description of what will be demonstrated]

### Key Highlights

* [Highlight 1]
* [Highlight 2]
* [Highlight 3]

### Demo Link/Details

[Link to demo, video, or instructions for accessing the demo]
```

## Information Extraction Guide

### From Sprint Planning Meeting Notes

**For "What Was Done This Sprint" Slide:**

**Completed Work:**
- Look for: Action items that were completed, features that shipped
- Extract: Initiative names, completion status, results
- Format: As bullet points with brief descriptions

**Key Deliverables:**
- Look for: "Shipped", "Completed", "Released" indicators
- Extract: Specific deliverables, features, or improvements
- Format: As a list of deliverables

**Metrics/Results:**
- Look for: Numbers, percentages, test results
- Extract: Key performance indicators, test outcomes
- Format: Include with context (e.g., "15% higher save rate")

**For "What's Coming Up Next Sprint" Slide:**

**Sprint Goal:**
- Look for: "Sprint goal" in Sprint Planning Decisions section
- Extract: The stated sprint goal
- Format: As a clear statement

**Planned Work:**
- Look for: "Next Steps / Action Items", "Sprint Planning Decisions"
- Extract: Initiatives starting, work planned, spikes
- Format: As bullet points with expected outcomes

**Focus Areas:**
- Look for: "Project Overview", "Technical Architecture Discussion"
- Extract: Key areas of focus, technical work, investigations
- Format: As a list of focus areas

**For "Demo" Slide (if applicable):**

**What to Demo:**
- Look for: Completed features, new capabilities, improvements
- Extract: Features that are ready to showcase
- Format: Brief description with key highlights

### From Other Information Sources

**From PRDs:**
- Extract: Feature descriptions, user flows, expected outcomes
- Use: To provide context for what was built or what's planned
- Format: As brief descriptions in slide content

**From Metrics Dashboards:**
- Extract: Results, performance data, test outcomes
- Use: To add quantitative results to "What Was Done" slide
- Format: Include numbers with context

**From Other Meeting Notes:**
- Extract: Decisions, blockers resolved, dependencies
- Use: To provide additional context for completed or planned work
- Format: As relevant bullet points

## Tips

1. **Keep slides concise** - Each slide should be scannable in 30 seconds
2. **Use bullet points** - Easy to read and present
3. **Include metrics** - Numbers and results make updates more impactful
4. **Be specific** - Avoid vague statements, include concrete deliverables
5. **Match sprint goal** - Ensure planned work aligns with stated sprint goal
6. **Only include demo slide** - If there's something meaningful to showcase
7. **Use consistent formatting** - Makes the presentation look professional
8. **Include links** - When referencing dashboards, PRDs, or other resources

## Common Phrases for Slides

**Completed Work:**
- "Shipped" / "Released" / "Completed"
- "Ready for launch" / "Ready to go live"
- "Finalized" / "Delivered"

**Planned Work:**
- "Kicking off" / "Starting"
- "Continuing work on"
- "Focusing on"
- "Investigating" / "Spiking"

**Metrics:**
- "X% increase/decrease"
- "X% higher/lower than"
- "Achieved X%"
- "Seeing X%"
